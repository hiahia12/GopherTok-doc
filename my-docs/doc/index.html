    亮点

    高可用

    服务的api和rpc启动多个docker实例，api使用traefik负载均衡，rpc通过etcd`实现负载，保证服务的可靠性，高峰时期可以轻松扩容

    高并发

    调用各个服务的rpc时采用并发调用，大大减少了响应时间; 讲高频率的数据采用redis作缓存，减少了mysql压力; 使用kafka异步写入mysql，增加系统吞吐量

    高性能

    kafka采用聚集写入，大大减少磁盘io和网络io
